📋 What Is Iron Vault?

Iron Vault is a singleton data provider (shared/iron_vault.py) that serves as the single access point for all options and market data in the PilotAI system. It wraps HistoricalOptionsData in offline mode — meaning it only reads from the local SQLite cache, never makes live Polygon API calls during backtests.

Key Principle: Cache miss → skip trade (return None). Cache miss → fabricate price = CRITICAL BUG

🔧 Usage

In Code

from shared.iron_vault import IronVault, IronVaultError

# Get the singleton (validates DB on first call)
hd = IronVault.instance()

# Pass to Backtester — drop-in replacement for HistoricalOptionsData
bt = Backtester(config, historical_data=hd)

# Coverage check
report = hd.coverage_report()
print(f"Contracts: {report['contracts_total']:,}")
print(f"Daily bars: {report['daily_bars_total']:,}")

🚀 Setup From Scratch

Step 1: Set API Key

echo "POLYGON_API_KEY=your_key_here" >> .env

Step 2: Validate Setup

python scripts/iron_vault_setup.py --verbose

This checks: API key presence, DB existence/size, per-ticker coverage, and flags gaps.

Step 3: Backfill Missing Data

# SPY (main backtest ticker)
python scripts/fetch_polygon_options.py --ticker SPY --resume

# Bulk daily bar backfill (fast — ~2 min)
python scripts/backfill_polygon_cache.py --workers 8

# Sector ETFs (for COMPASS portfolio backtests)
python scripts/fetch_sector_options.py --ticker XLE --resume

Step 4: Run a Backtest

python scripts/run_optimization.py --config configs/champion.json
There is no --heuristic flag. All backtests use real data. Always.
