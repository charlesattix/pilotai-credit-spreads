Found by: CC1

Alert.from_opportunity() hardcodes risk_pct=0.02 for every alert. The risk gate checks this fake number, not the actual position size. Risk rules 1 & 2 are effectively enforcing wrong limits.

⚠️ HIGH SEVERITY ISSUES

8. Iron Condor Execution is Non-Atomic HIGH

Found by: CC1

ICs are submitted as two separate 2-leg orders. If the put wing fills but call wing fails, you get a naked bull put spread. Cancel-after-fail may arrive too late.

9. Correlated Position Check Never Matches HIGH

Found by: CC1

Risk rule #5 compares direction.value ("bullish"/"bearish") against DB values ("bull_put"/"bear_call"). These never match → rule is effectively disabled.

10. IC Max Loss Doubles Actual Risk HIGH

Found by: CC1

Position sizer calculates IC max loss as (spread_width × 2 - credit) × 100, treating both wings as losing simultaneously (impossible). ICs get ~half the contracts they should.

11. No Default config.yaml HIGH

Found by: CC1

Running python main.py scan with no --config crashes on file-not-found. Only experiment-specific configs exist.

12. Risk Gate Bypassed When Alpaca Down HIGH

Found by: CC2

When Alpaca API fails, fallback account_state has empty positions → all risk checks pass (no existing exposure detected). Could allow unlimited new positions.

13. No SQLite Busy Timeout HIGH

Found by: CC2

Scanner and PositionMonitor write to same DB concurrently. Default busy timeout is 0 (fail immediately). Concurrent writes will get SQLITE_BUSY with no retry.

14. Profit Target May Be Unreachable HIGH

Found by: CC1

If profit_target_pct is stored as 55 (percent) not 0.55 (decimal), the monitor converts it to 5500%, making profit target unreachable. Positions would only exit via stop loss or DTE.

🎯 Verdict & Recommendations

SPY-Only Experiments (exp036, exp059, exp154): CAN paper trade with caveats. The core pipeline works. Fix issues #7 (fake risk_pct), #9 (correlated check), #14 (profit target conversion) before going live. The remaining issues are tolerable for paper trading.
COMPASS Experiment (exp305): CANNOT paper trade. The COMPASS features (dynamic sectors, macro scoring, RRG filtering, portfolio allocation, risk rules 8-10) exist only in the backtester. The live scanner has none of this code. The technical report describes planned features as if they're implemented — they are not.
