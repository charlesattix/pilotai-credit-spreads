Setup (Same for Both — 5 Minutes)

Step 1: Clone and checkout

git clone git@github.com:charlesattix/pilotai-credit-spreads.git
cd pilotai-credit-spreads
git checkout maximus/champion-config
pip install -r requirements.txt

Step 2: Create environment file

Create one .env file per experiment you want to run:

EXP-400: .env.champion

ALPACA_API_KEY=your_paper_key
ALPACA_API_SECRET=your_paper_secret
POLYGON_API_KEY=your_polygon_key
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id

EXP-401: .env.exp401

ALPACA_API_KEY=your_paper_key
ALPACA_API_SECRET=your_paper_secret
POLYGON_API_KEY=your_polygon_key
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
⚠️ Use Alpaca PAPER trading keys, not live. Both configs have paper_mode: true which blocks live API URLs, but use paper keys as an extra safety layer.

Step 3: Smoke test (dry run — no trades placed)

# EXP-400
python main.py scan --config configs/paper_champion.yaml --env-file .env.champion

# EXP-401
python main.py scan --config configs/paper_exp401.yaml --env-file .env.exp401

Verify: ✅ Alpaca connects, ✅ Polygon data loads, ✅ Regime detected, ✅ No errors

Step 4: Start paper trading

# EXP-400
python main.py scheduler --config configs/paper_champion.yaml --env-file .env.champion

# EXP-401
python main.py scheduler --config configs/paper_exp401.yaml --env-file .env.exp401

Step 5: Run in background (recommended)

tmux new-session -d -s exp400 \

What Happens After You Start

  • System scans 14 times per day during market hours (9:30 AM – 4:00 PM ET, weekdays)
  • Opens trades when entry signals fire (credit spreads, iron condors, or straddle/strangles depending on experiment)
  • Monitors positions for profit target / stop loss exits
  • Sends Telegram alerts on every entry and exit (if configured)
  • Each experiment has its own isolated database — they don't interfere

How EXP-401 Works (The Blend)

Two strategies that complement each other:

Credit Spreads (12% risk/trade)

  • Bull market → sell bull put spreads
  • Bear market → sell bear call spreads
  • Regime-adaptive via 80-day MA
  • Scales DOWN in bear/high-vol (0.3×)

Straddle/Strangles (3% risk/trade)

  • Post-event short strangles (FOMC, CPI)
  • Profits from IV crush
  • Scales UP in high-vol (2.5×)
  • Hedges the CS component in bear markets
Why this works: When credit spreads scale down in volatile markets, straddle/strangles scale up — and vice versa. In 2022 (SPY -19.4%), the CS component lost money but the S/S component gained enough to finish the year at +2.2%. Low correlation = smoother equity curve.

When to Worry

🔴 HARD STOP if:

  • Drawdown exceeds 30%
  • 5+ consecutive losses
  • System enters trades that don't match the regime
  • Any execution error that places unintended orders

🟡 PAUSE after 4 weeks if:

  • Win rate below 60% (expect ~80%)
  • Annualized return tracking below 10%
  • Trade count 50%+ off from expectations

🟢 SUCCESS after 8 weeks:

  • Win rate >70%
  • Positive cumulative P&L
  • Max drawdown <20%
  • → Ready to discuss live capital
