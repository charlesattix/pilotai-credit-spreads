AMBER FLAG 1: Weekly-loss breach reduction removed from legacy sizer
The _legacy_size() method had its weekly_loss_breach *= 0.5 cut removed because "backtester has neither." This is correct for alignment, but it removes a safety brake. The flat-risk sizer also lost it. Ensure the drawdown CB (which IS aligned with backtester) compensates. Risk: medium — CB catches large drawdowns, but weekly breaches are more granular.
AMBER FLAG 2: Stop-loss simplified — spread-width cap removed
position_monitor.py previously had a dual stop-loss: formula-based (1+mult)×credit AND a 90%-of-spread-width safety cap. The spread-width cap was removed for backtester alignment. For narrow spreads with high credit (e.g., $2 credit on $3 spread), the formula threshold (1+3.5)×2 = $9 exceeds the $3 max spread width — the stop-loss can never fire. This is actually fine because the spread can't trade above its width, but the previous code's defensiveness against pricing errors is lost.
AMBER FLAG 3: Hardcoded market holidays through 2030
position_monitor.py now has hardcoded holiday dates through 2030 (60+ dates). This works but is brittle. Consider using a library like exchange_calendars or a dynamic calendar. Not a blocker.
POSITIVE: Paper-mode safety check added
_validate_paper_mode_safety() in main.py prevents paper_mode configs from accidentally hitting live Alpaca. Checks both alpaca.paper flag and base_url for "paper" substring. Excellent defensive coding.
POSITIVE: IC partial close escalation
3-attempt retry with 5s delay, CRITICAL log, DB flag, and position left in pending_close to prevent auto-retry loops. Manual intervention required. This is exactly the right approach for a 4-leg position where partial fills can create naked exposure.
